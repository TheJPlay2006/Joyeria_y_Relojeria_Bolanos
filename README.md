# Joyería y Relojería Bolaños

Sistema web desarrollado para apoyar la gestión administrativa de la **Joyería y Relojería Bolaños**, permitiendo organizar productos, inventario, ventas, clientes y reparaciones de una forma más clara, segura y eficiente.

## Descripción del proyecto

Este proyecto nace como una solución para mejorar los procesos internos de la Joyería y Relojería Bolaños, un negocio dedicado a la confección, reparación y venta de joyas y relojes.

Actualmente, muchos procesos se realizan de forma manual mediante cuadernos, recibos físicos, llamadas y mensajes por WhatsApp. Esto puede provocar pérdida de información, atrasos en la búsqueda de datos, dificultad para dar seguimiento a trabajos pendientes y poco control del inventario.

El sistema busca centralizar la información del negocio en una plataforma web sencilla, permitiendo consultar, registrar y actualizar datos importantes de manera más rápida.

## Objetivo general

Desarrollar un sistema web para la Joyería y Relojería Bolaños que permita mejorar la gestión de productos, inventario, ventas y seguimiento de reparaciones, facilitando el control de la información y los procesos administrativos del negocio.

## Funcionalidades principales

- Inicio y cierre de sesión.
- Gestión de usuarios y roles.
- Registro, edición y consulta de productos.
- Control de inventario.
- Alertas de stock mínimo.
- Registro de clientes.
- Búsqueda de clientes por nombre o teléfono.
- Registro de órdenes de trabajo.
- Seguimiento de reparaciones.
- Cambio de estado de trabajos:
  - Pendiente
  - En proceso
  - Finalizado
  - Cancelado
- Registro de ventas.
- Consulta de reportes.
- Dashboard principal con información relevante del negocio.

## Roles del sistema

### Administrador

Puede gestionar usuarios, productos, inventario, clientes, ventas, reparaciones y reportes.

### Empleado

Puede registrar y consultar información relacionada con clientes, ventas y reparaciones, según los permisos asignados.

## Tecnologías y herramientas

- Visual Studio Code
- PostgreSQL
- XAMPP
- GitHub
- Navegador web

## Metodología de desarrollo

El proyecto se desarrolla utilizando la metodología **Cascada**, siguiendo un proceso ordenado por etapas:

1. Análisis de requerimientos.
2. Diseño de base de datos e interfaces.
3. Implementación de módulos.
4. Pruebas funcionales.
5. Documentación final.

## Alcance del sistema

El sistema contempla:

- Gestión de productos.
- Gestión de inventario.
- Gestión de clientes.
- Registro de ventas.
- Registro y seguimiento de reparaciones.
- Reportes básicos.
- Notificaciones internas o registro de comunicación con clientes.

## Limitaciones

Este proyecto no incluye inicialmente:

- Aplicación móvil.
- Facturación electrónica.
- Pasarela de pagos.
- Integración bancaria.
- Tienda en línea.
- Integración con sistemas externos.
- Compra directa de productos desde el sistema.

## Requerimientos no funcionales principales

- Interfaz sencilla e intuitiva.
- Diseño responsivo para uso desde navegador.
- Acceso protegido mediante usuario y contraseña.
- Manejo seguro de información de clientes.
- Buen rendimiento en búsquedas.
- Código organizado y mantenible.
- Posibilidad de agregar nuevos módulos en el futuro.
